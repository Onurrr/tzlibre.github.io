Introduction

Whitelisting is the first step to split XTZ coins and get TZL coins. Contributors who whitelist their own address will be allowed to split their XTZ and receive TZL pre-launch tokens. Contributors who don't whitelist their address will not be able to split.

Who

All Tezos ICO contributors can whitelist.

When

The whitelist is now open. Our community is currently discussing whether to close the whitelist before splitting begins to reward early adopters, or to keep whitelisting open for longer to accomodate late adopters. We'll announce on our Telegram channel when the whitelist has closed.

How it works

Whitelisting is a trustless and pseudonymous process based on an Ethereum smart contract. During whitelisting three tasks are executed: proving ownership of a DLS-Tezos ICO contribution address, associating it with an Ethereum address, signing a pseudonymous declaration.

Methods

We suggest to manually follow the protocol. If you're not an expert you can use the command line tools. If you use the command line tool we encourage you to at least read the source code before executing it.

Next steps

After whitelisting we'll issue a pre-launch TZL token on the Ethereum blockchain. TZL tokens will only be credited to whitelisted Ethereum addresses. Following the EOS example, pre-launch tokens will be converted into native coins token when TzLibre mainnet launches. We're currently ugrading the splitting process, and will post updates on our public Telegram channel.

Security

You should only handle sensitive data, such as contribution data, on a machine that is not compromised. You can also disconnect your machine from any network while handling sensitive contribution data. If you're not certain about the security of your machine you should use a physically airgapped one.

Approach

The whitelist procedure is completely trustless and consists of the invocation of the Ethereum smart contract method whitelist of the TzLibreRegistry Ethereum smart contract with three parameters:

  • the signature of the Ethereum address made through the Tz private key
  • the signature of a declaration made through the Tz private key
  • the Tz public key, to verify signatures

Smart contract

In order to split your Libre tezzies (TZL) you must whitelist an ETH address under your control and prove ownership of the DLS-Tezos private key whose funds you're going to split into Libre tezzies.This is done by providing a couple of digital signatures to the TzLibreRegistry Ethereum smart contract. The Solidity code of the registry contract is available on GitHub and deployed on the Ethereum Mainnet at the address [ TODO: 0x... ADD_LINK_TO_ETHERSCAN ] Ropsten at the address 0x183601C3d1B9e421CBE789843e8312De30700C6a. This contract therefore assigns the correct amount of Libre tezzies (at a ratio of one to one with DLS tezzies got during ICO)

Oracle

Due to the limitations of Ethereum smart contracts, the procedure resorts to Oraclize, an external oracle. This platform verifies the signature and contribution to the DLS-Tezos ICO using public data from the BTC and ETH blockchains. The source code of the Docker container run by Oraclize is available on GitHub and is deployed on IPFS at

Oraclize produces integrity proofs to prove that computation has been executed correctly. You can check the proofs produced by Oraclize at http://app.oraclize.it/service/monitor or download their verification tool.

Protocol

Following this protocol you can create a verified association between an ETH address you control (and where you'll get Libre tezzies) and your TzLibre address (same as Tz address).

Definitions

The protocol employs:

  • TzLibre public and private keys are the same as in DLS-Tezos.
  • Ed25519 as the digital signature algorithm (Ed25519_sign)
  • Blake2b as the hashing algorithm (Blake2b_64 which produces a 64 bytes long output)

Method info

The goal of the whitelisting protocol is to invoke the method whitelist of the TzLibreRegistry Ethereum smart contract (deployed on the Ethereum Mainnet at [ TODO: 0x...] Ropsten at the address 0x183601C3d1B9e421CBE789843e8312De30700C6a).

The method signature is whitelist(bytes32,bytes,bytes) and its ABI is:

{
  "constant": false,
  "inputs": [
    {
      "name": "TZL_pk",
      "type": "bytes32"
    },
    {
      "name": "ETH_addrSignature",
      "type": "bytes"
    },
    {
      "name": "declarationSignature",
      "type": "bytes"
    }
  ],
  "name": "whitelist",
  "outputs": [],
  "payable": true,
  "stateMutability": "payable",
  "type": "function"
}

1. Generate parameters

The whitelist method accepts three parameters:

  • TZL_pk: the TzLibre public key
  • ETH_addrSignature: the signature of an Ethereum address of your choice made through the TzLibre private key
  • declarationSignature: the signature of a declaration statement made through the TzLibre private key

TZL_pk

If you already know your DLS-Tezos private (aka "secret") and public key, you can skip this step. Otherwise you can generate them from the following information:

  • mnemonic: 15 words mnemonic seed that you find it in the PDF document provided during the DSL-Tezos ICO
  • email: you find it in the PDF document provided during the DSL-Tezos ICO
  • password: you should remember it or have it written down somewhere

Once you have all the required data you can follow these steps (links refer to the fundraiser-tools by official DLS-Tezos GitHub repository):

seed = f(mnemonic, email, password)
TZL_pk, TZL_sk = Ed25519_keys_generation(seed)

ETH_addrSignature

You now must sign ETH_addr (an ETH address you control) with your TZL secret key TZL_sk:

ETH_addrHash = Blake2b_64(ETH_addr)
ETH_addrSignature = Ed25519_sign(ETH_addrHash, TZL_sk)

declarationSignature

Finally, you have to sign the hash of the following declaration. This serves as a pseudonymous way to ask Tezos Stiftung to support TzLibre development:

I hereby cryptographically prove to be a contributor of Tezos Stiftung (CHE-290.597.458), a Swiss Foundation based in Gubelstrasse 11, 6300 Zug, Switzerland. I recognize and welcome the existence multiple implementations of Tezos. I ask and expect Tezos Stiftung to foster competition among them by funding and supporting their development, marketing and growth. Funds allotted to various Tezos implementations shall always be directly proportional to their market capitalization at the time of each distribution of funds. Distribution of funds to multiple existing Tezos implementations shall begin no later than January 1st 2019 and consistently continue throughout time. Following priorities autonomously set by each community, Tezos Stiftung shall distribute funds in the most appropriate, effective and transparent way.

You have to sign it with your TZL secret key TZL_sk:

declarationHash = Blake2b_64(declaration)
declarationSignature = Ed25519_sign(declarationHash, TZL_sk)

Command-line tools

The Command-line tools is a set of 4 tools aimed at helping reconstruct your DLS-Tezos private key and use it to follow the whitelisting procotol, that is associating your Ethereum address and signing a pseudonymous declaration.

You should use the Command-line tools only if you're not able to produce signatures by manually following the whitelisting protocol.

Command-line tools on GitHub

In line with original blockchain ethos we strongly encourage you to read the source code of all tools before executing them.

Estimated time

30 minutes

Required skills

Bash, Node.js, Python

Description

The 4 Command-line tools are:

  • Dockerfile: it allows to setup a sandboxed environment. This increases safety and stability, and allows the user to avoid installing dependencies and configuring the environment.
  • gen-signatures.sh (Bash script): this script manages the i/o of the whole process. It launches both tools, and feeds the output of the first tool into the second one.
  • pykeychecker (Python): this is a working fork of DLS "fundraiser tools", which are currently broken. We fixed them, and then added the option of signing an Ethereum address and a pseudonymous declaration. You can verify the code diff between the two versions.
  • jstxencoder (Node.js): this tool generates Ethereum transaction data by encoding parameters. It takes in input TZL_pk, ETH_addrSignature, declarationSignature and correctly encodes them into an Ethereum transaction data.

1. Generate tx data

Follow the instructions in the Readme file, and save the resulting transaction data.

2. Send tx

Use the generated transaction data to send an Ethereum transaction with these parameters:

  • from: your Ethereum address (ETH_addr)
  • to: TzLibreRegistry contract address 0x183601C3d1B9e421CBE789843e8312De30700C6a (Ropsten) [TODO update in production]
  • tx data: [ output of step 1 ]
  • value: 0.03 ETH (oracle fees)
  • gas: 1000000
  • gasPrice: 20 GWei (suggested)

3. Verify

After your transaction has been mined, you should wait ~5 more minutes for Oraclize to provide its response. You then must verify that your Ethereum address has been successfully whitelisted. You can do it with our verification tool.

4. Follow updates

If whitlisting was successful, you should follow our public Telegram channel for updates on splitting.
